# PJMTG-automation

会議運営を自動化する Google Apps Script (GAS) プロジェクトです。
1つの Google スプレッドシートを「制御盤」として、以下の一連の流れを自動化します。

1. **会議のスケジュールを決定** — 「会議予定」シートに1行登録すると、Googleカレンダー + Meet会議を自動作成
2. **時間計測ツールで開始打刻** — Meetの実際の参加ログ(入室〜退室時刻)をもとに、**実際に参加した人だけ**を既存の社内活動タイマー(`ActivityTimer.gs`)のメンバーシートに実績時間で自動打刻
3. **議事録機能・録音機能をON** — 会議作成時に Meet の自動録画・自動文字起こし・スマートノート(議事録)をON
4. **会議終了後、Google Chatに議事録と録画を投稿** — Meet APIで生成物が揃い次第、指定のChatスペースに投稿
5. **議事録を既読の人がスタンプを押す** — 投稿メッセージへのリアクションを収集し「既読」とみなす
6. **参加率・既読率の分析** — 招待者・実参加者(Meet参加ログ)・既読者を突き合わせて「参加分析」シートに集計

## ファイル構成

```
src/
  appsscript.json          マニフェスト(スコープ・APIの設定)
  Utils.gs                 共通ユーティリティ、シート定義
  00_Setup.gs               初期化(シート作成)
  01_ScheduleMeeting.gs     ① 会議作成(カレンダー + Meet)
  02_TimeTracking.gs        会議ステータスの自動遷移(作成済み→開催中→終了)
  03_MeetingArtifacts.gs    ③ Meet API(自動録画/文字起こし/議事録ON、会議記録・参加ログ取得)
  04_PostToChat.gs          ④ Google Chat への投稿
  05_ReactionTracking.gs    ⑤ リアクション(スタンプ)収集
  06_AttendanceAnalysis.gs  ⑥ 出席率/既読率の分析
  07_AttendanceStamping.gs  ② Meet参加ログに基づく実績打刻(ActivityTimer.gsの関数を呼び出す)
  ActivityTimer.gs          既存の社内活動打刻タイマー(メンバー/グループ管理、手動修正、サマリー等)
  Main.gs                   メニュー、トリガー一括設定
```

`ActivityTimer.gs` は既存の社内活動タイマーツールをそのまま取り込んだものです。
`onOpen()` はプロジェクト内で1つしか定義できないため、メニュー構築部分だけ
`buildActivityTimerMenu_()` という名前に変更し、`Main.gs` の `onOpen()` から呼び出す形にしています。
それ以外の挙動(グループ管理・手動修正・年度リセット・スマホ打刻など)は変更していません。

## 前提条件

- Google Workspace(自動録画・自動文字起こし・スマートノートが利用できるエディション。録画/文字起こしはBusiness Standard以上、スマートノートはGeminiアドオンが必要な場合があります。管理者が Google 管理コンソールでこれらの機能を有効化している必要があります)
- スクリプトを紐づける Google スプレッドシート — **既存の「活動タイマー」を使っているスプレッドシートと同じもの**に、本プロジェクトのシート・スクリプトを追加する
- 「活動タイマー」側で、打刻対象となる各メンバーの `メンバー_◯◯` シートが作成済みであること(未作成なら「⏱ 活動タイマー」→「⚙ 初期セットアップ」または「➕ メンバーを追加」で作成)
- 管理者操作(メンバー名変更・削除・シート保護・年度リセット)を使う場合は、スクリプトプロパティ `ADMIN_PASSWORD` を設定しておくこと
- スクリプトの実行アカウントが、投稿先の Google Chat スペースのメンバーであること

## セットアップ手順

### 1. Apps Script プロジェクトの作成

1. 対象の Google スプレッドシートを開き、「拡張機能」→「Apps Script」を選択
2. 表示されたプロジェクトに、`src/` 配下の各ファイルの内容をコピーして貼り付ける
   (もしくは [clasp](https://github.com/google/clasp) を使う場合は下記「clasp を使う場合」を参照)
3. `appsscript.json` の内容を反映させる(Apps Scriptエディタの「プロジェクトの設定」→「"appsscript.json" マニフェストファイルをエディタで表示する」を有効化してから編集)

### 2. Google Cloud プロジェクトでAPIを有効化

Apps Script に紐づく Google Cloud プロジェクトで、以下のAPIを有効化してください。

- Google Calendar API
- Google Chat API
- Google Meet API (`meet.googleapis.com`)

### 3. Chat スペースIDの確認

投稿先の Google Chat スペースの ID(`spaces/xxxxxxxx` の形式)を控えます。
スペースのURL、または Chat API の `spaces.list` から取得できます。

### 4. 初期化

1. スプレッドシートを再読み込みすると、メニューに「会議自動化」が追加されます
2. 「① 初期化(シート作成)」を実行 → 必要なシートが自動作成されます
3. 「設定」シートの `ChatスペースID` と `カレンダーID` を実際の値に書き換える
4. 初回実行時にOAuth権限の承認画面が出るので許可する
5. 「② 定期実行トリガーを設定」を実行 → 各処理が自動実行されるようになります

### 5. 会議の登録

「会議予定」シートに1行追加するだけで自動化が始まります。

| 会議名 | 開始日時 | 終了日時 | 出席者(カンマ区切りメール) | 打刻対象メンバー(氏名カンマ区切り) | 会議ID | 会議コード | 会議記録ID | MeetURL | ステータス | 実績打刻済み |
|---|---|---|---|---|---|---|---|---|---|---|
| 定例MTG | 2026/07/15 10:00 | 2026/07/15 11:00 | a@example.com, b@example.com | 田中, 鈴木 | (自動入力) | (自動入力) | (自動入力) | (自動入力) | (自動更新) | (自動更新) |

「出席者(カンマ区切りメール)」はカレンダー招待・出欠確認用、
「打刻対象メンバー(氏名カンマ区切り)」は「この会議では誰の出席を打刻の対象にするか」の
候補リストです。**氏名は活動タイマー側の「メンバー_◯◯」シート名と完全一致させてください**
(実際に打刻されるのはこのうちMeetに実際参加した人だけです。詳しくは次のセクション)。

ステータスは `(空)→作成済み→開催中→終了→投稿済み→分析済み` と自動で遷移します
(このステータス自体は時刻だけで進み、打刻とは連動しません)。

## 出席者だけを打刻する仕組み(重要)

打刻は「打刻対象メンバー」全員に一律で入るのではなく、**Meetの実際の参加ログ
(入室〜退室時刻)と一致した人だけ**に、実際の参加時間で入ります(`07_AttendanceStamping.gs`)。
会議終了後、Meet側の参加記録が確定し次第(10分おきに自動リトライ)、活動タイマーの
メンバーシートに1行追記される形です。ライブでの「開始と同時に打刻」ではなく、
「終了後にまとめて、実際の参加時間で正確に打刻」する方式である点に注意してください。

### なぜ相関表が必要か

Meet APIは参加者をメールアドレスや氏名ではなく `users/xxxxxxxx` というGoogleアカウントの
リソースIDでしか返しません(Chat APIのリアクション取得者も同様)。そのため
「ユーザーID対応表」シートで `users/xxxxxxxx → メールアドレス → 氏名(活動タイマー用)` の
対応を管理しています。未知のユーザーIDが検出されると自動で行が追記されるので、
社内の管理者は初回だけ「氏名(活動タイマー用)」欄(リアクションのメールアドレス解決用途も
兼ねる場合はメールアドレス欄も)を埋めれば、以降は自動的に解決されます。

### 制約・注意点

- 電話参加や匿名/ゲスト参加(Googleアカウントを持たない参加者)は自動で紐付けできません。手動修正機能で追記してください。
- 退室→再入室した場合、その間の空白時間は引かれず「最初の入室〜最後の退室」の幅で計算されます。
- 「会議予定」シートの「実績打刻済み」列がTRUEになっている行は、その会議についてもう一度は打刻されません(二重打刻防止)。

## 動作タイミングに関する注意

- 会議終了直後は Meet 側の会議記録・参加ログ・録画/文字起こし/議事録の生成に数分〜十数分かかります。
  `stampActualAttendanceForFinishedMeetings` と `postFinishedMeetingsToChat` はどちらも
  生成が完了するまで自動的に再試行し続けます(10分おき)。
- リアクション集計・参加率分析は1時間おきに実行されるため、即時反映ではありません。
  急ぎ確認したい場合はメニューから手動実行できます。

## clasp を使う場合(任意)

```bash
npm install -g @google/clasp
clasp login
cp .clasp.json.example .clasp.json
# .clasp.json の scriptId を実際のスクリプトIDに書き換える
clasp push
```

## 手動実行

スプレッドシートのメニュー「会議自動化」から、各ステップを個別に手動実行することもできます。
「一連の処理を今すぐ全部実行」を選ぶと、会議作成・ステータス更新・実績打刻・Chat投稿・
リアクション収集・参加率分析をまとめて1回実行します。
